## 課程預覽

今天練習日本「プロジェクト管理」最常遇到的 5 個情境：

1. 定例会：開場、アジェンダ、時間管理
2. 期限調整：タスク担当と締切の再確認
3. 相談：詰まり（ブロッカー）と支援依頼
4. 変更管理：仕様変更の影響、優先度、決裁
5. 会後：議事録、TODO、次回日程

## 情境 1: 定例会の開始（アジェンダ確認）

![定例會議](https://images.unsplash.com/photo-1542744173-8e7e53415bb0?w=800&h=400&fit=crop&crop=center)
<!-- Image Prompt: Japanese project status meeting diagram - agenda sheet, timer clock, participants around table, Japanese labels (定例会, アジェンダ, 進捗, 課題), modern meeting room, educational diagram style, no text overlays, only Japanese labels positioned at corresponding elements -->

<div style="text-align: left">

**司会（あなた）:** それでは定例会を始めます。本日のアジェンダは、①進捗、②課題、③今週の優先事項の三つです。終了は30分後を予定しています。<br>
(soredewa teireikai o hajimemasu. honjitsu no ajenda wa, ichi shinchoku, ni kadai, san konshuu no yuusen jikou no mittsu desu. shuuryou wa sanjuppun go o yotei shite imasu.)<br>
那麼開始例行會議。今天議程是 ①進度 ②課題 ③本週優先事項三項。預計 30 分鐘後結束。<br>
**Let’s start the regular meeting. Today’s agenda is (1) progress, (2) issues, and (3) this week’s priorities. We plan to finish in 30 minutes.**<br>

</div>

<br>

<div style="text-align: right">

**メンバー:** 承知しました。共有資料はこのスライドで合っていますか？<br>
(shouchi shimashita. kyouyuu shiryou wa kono suraido de atte imasu ka?)<br>
了解。共用資料是這份投影片對嗎？<br>
**Understood. Is this the correct shared slide deck?**<br>

</div>

### 重點句型
- それでは始めます。(soredewa hajimemasu) - 那就開始
- 終了は〜後を予定しています。(shuuryou wa ~ go o yotei shite imasu) - 預計……後結束

### 相關單字
- 定例会 (teireikai) - 例行會議
- 進捗 (shinchoku) - 進度
- 優先事項 (yuusen jikou) - 優先事項

## 情境 2: 期限と担当の再確認（やり切るために）

![期限與分工](https://images.unsplash.com/photo-1553877522-43269d4ea984?w=800&h=400&fit=crop&crop=center)
<!-- Image Prompt: Japanese task deadline ownership diagram - checklist, calendar deadline, owner name tag, Japanese labels (担当, 締切, 期限, 依頼), clean infographic style, educational diagram style, no text overlays, only Japanese labels positioned at corresponding elements -->

<div style="text-align: left">

**あなた:** 念のため確認です。機能Aのテストは山田さん担当で、締切は木曜の17時、で合っていますか？<br>
(nen no tame kakunin desu. kinou A no tesuto wa Yamada-san tantou de, shimekiri wa mokuyou no juushichiji, de atte imasu ka?)<br>
為了確認：功能 A 的測試是山田負責，截止週四 17:00，這樣對嗎？<br>
**Just to confirm: Yamada owns testing for Feature A, and the deadline is Thursday 5 p.m.—is that correct?**<br>

</div>

<br>

<div style="text-align: right">

**山田:** はい、合っています。ただ、環境が不安定なので、最悪の場合は金曜午前にずれ込むかもしれません。<br>
(hai, atte imasu. tada, kankyou ga fuantei nanode, saiaku no baai wa kinyou gozen ni zurekomu kamoshiremasen.)<br>
是的沒錯。但環境不穩，最糟可能延到週五上午。<br>
**Yes, that’s correct. But the environment is unstable, so worst case it may slip to Friday morning.**<br>

</div>

### 重點句型
- 念のため確認です。(nen no tame kakunin desu) - 為了確認
- 最悪の場合は〜かもしれません。(saiaku no baai wa ~ kamoshiremasen) - 最糟可能……

### 相關單字
- 担当 (tantou) - 負責
- 締切 (shimekiri) - 截止
- ずれ込む (zurekomu) - 延後

## 情境 3: 詰まりの相談（ブロッカーと支援）

![阻塞相談](https://images.unsplash.com/photo-1522071820081-009f0129c71c?w=800&h=400&fit=crop&crop=center)
<!-- Image Prompt: Japanese blocker support request diagram - warning icon, dependency chain, helper hand icon, Japanese labels (詰まり, 依存, 支援, 相談), educational diagram style, no text overlays, only Japanese labels positioned at corresponding elements -->

<div style="text-align: left">

**メンバー:** すみません、開発が詰まっています。APIの仕様が未確定で、実装に入れません。いつ確定しますか？<br>
(sumimasen, kaihatsu ga tsumatte imasu. API no shiyou ga mikakutei de, jissou ni hairemasen. itsu kakutei shimasu ka?)<br>
不好意思，開發卡住了。API 規格還沒定，無法開始實作。何時會定案？<br>
**Sorry—I’m blocked. The API spec isn’t finalized, so I can’t start implementation. When will it be finalized?**<br>

</div>

<br>

<div style="text-align: right">

**あなた:** 失礼しました。今日中に確定できるよう、担当チームに優先で確認します。暫定版でも進められる範囲を切り出してもらえますか？<br>
(shitsurei shimashita. kyoujuu ni kakutei dekiru you, tantou chiimu ni yuusen de kakunin shimasu. zanteiban demo susumerareru han'i o kiridashite moraemasu ka?)<br>
抱歉。為了今天內能定案，我會優先向負責團隊確認。你可以先把即便暫定版也能前進的範圍切出來嗎？<br>
**My apologies. I’ll prioritize checking with the owning team so we can finalize today. In the meantime, could you carve out what you can proceed with even using a provisional spec?**<br>

</div>

### 重點句型
- 詰まっています。(tsumatte imasu) - 卡住了
- 今日中に〜できるようにします。(kyoujuu ni ~ dekiru you ni shimasu) - 我會確保今天內……

### 相關單字
- 仕様 (shiyou) - 規格
- 未確定 (mikakutei) - 未定案
- 暫定 (zantei) - 暫定

## 情境 4: 仕様変更の依頼（影響と優先度）

![需求變更](https://images.unsplash.com/photo-1556761175-b413da4baf72?w=800&h=400&fit=crop&crop=center)
<!-- Image Prompt: Japanese change request impact diagram - change request form, impact arrows to schedule and cost, approval stamp, Japanese labels (仕様変更, 影響, 工数, 決裁), clean diagram, educational diagram style, no text overlays, only Japanese labels positioned at corresponding elements -->

<div style="text-align: right">

**依頼側:** 仕様変更をお願いしたいです。画面に検索フィルターを追加できますか？<br>
(shiyou henkou o onegai shitai desu. gamen ni kensaku firutaa o tsuika dekimasu ka?)<br>
想拜託規格變更。畫面能加上搜尋篩選器嗎？<br>
**We’d like to request a spec change. Can we add a search filter to the screen?**<br>

</div>

<br>

<div style="text-align: left">

**あなた:** 可能です。ただ、工数が増えるので、リリース日への影響を確認させてください。優先度は「必須」ですか、それとも「できれば」ですか？<br>
(kanou desu. tada, kousuu ga fueru node, ririisu-bi e no eikyou o kakunin sasete kudasai. yuusendo wa \"hissu\" desu ka, soretomo \"dekireba\" desu ka?)<br>
可以。不過工數會增加，請讓我確認對發版日的影響。優先度是「必須」還是「可以的話」？<br>
**It’s possible. However, it will increase effort, so please let us confirm the impact on the release date. Is the priority “must-have” or “nice-to-have”?**<br>

</div>

<br>

<div style="text-align: right">

**依頼側:** できればで大丈夫です。次のリリースで入れられれば助かります。<br>
(dekireba de daijoubu desu. tsugi no ririisu de irerareba tasukarimasu.)<br>
可以的話就好。若能放到下一次發版會很有幫助。<br>
**Nice-to-have is fine. It would help if we can include it in the next release.**<br>

</div>

### 重點句型
- 影響を確認させてください。(eikyou o kakunin sasete kudasai) - 請讓我確認影響
- 優先度は〜ですか？(yuusendo wa ~ desu ka) - 優先度是……嗎？

### 相關單字
- 仕様変更 (shiyou henkou) - 規格變更
- 工数 (kousuu) - 工時／工數
- 影響 (eikyou) - 影響

## 文化小知識：日本のプロジェクト会話のポイント

1. 定例会は「進捗→課題→次の一手」の順で短く回すと、時間超過しにくいです。
2. 変更依頼は「できる／できない」より先に、**影響（工数・スケジュール）** と **優先度** を確認するのが一般的です。
3. 会後は議事録で「決定事項」「TODO」「期限」「担当」を明文化し、認識ずれを防ぎます。

## 小故事：詰まりを早めに出した日

佐々木さんはAPIが未確定で手が止まり、会議で「詰まっています」と正直に共有しました。司会のあなたが当日中の確定を取りに行き、暫定でも進められる部分を切り出したことで、翌日には作業が再開。さらに変更依頼も「必須かどうか」を確認して次リリースに回せたため、今週の締切は守れました。彼は「早めに言うほど、助けが早い」と実感しました。
